# 🔮 病毒预测器（Viral Oracle）

## 场景描述

用户写好一条推文后（还没发），点击按钮让 AI 预测：

```
🔥 病毒潜力评分：73/100

📊 预测模型：
  - 预计 likes：    200 ~ 800
  - 预计 retweets： 50 ~ 300
  - 预测回复数：    30 ~ 120

🧬 病毒因子分析：
  ✅ 情绪唤醒度：高（惊讶+愤怒混合）→ 最易传播的情绪组合
  ✅ 信息不对称：高（提出了反直觉观点）→ 引发讨论
  ⚠️ 话题争议性：中 → 会被讨论但不会撕裂
  ❌ 格式可读性：低 → 长段落，建议拆成 3 条 thread

💡 优化建议：
  1. 把第 2 句单独抽出做开头 hook
  2. 加一个具体数字增强可信度
  3. 结尾加「你怎么看？」提升互动率
```

## 为什么有趣

- **虚荣心**驱动——每个发推的人都想知道「这条能不能火」
- 有了预测，用户会**反复修改推文来追高分**，形成游戏化体验
- 预测不准的时候更有趣——「AI 说能火但只有 3 个 likes」，这本身就是个段子

## 技术实现

- 在推文编辑区域注入预测按钮
- 提取推文文本内容，通过 ChatClient 分析
- prompt 工程 + 历史数据训练
- 预测准确度不需要很高（60% 就够），关键是呈现方式让人觉得可信又好玩

## 交互设计要点

- 在 X 推文编辑框旁边插入「预测」按钮
- 评分用进度条/仪表盘呈现，视觉冲击力强
- 优化建议用可点击列表，方便逐项改进

## 优先级

P1 — 游戏化体验，用户粘性强
