# 💀 推文尸检报告（Tweet Autopsy）

## 场景描述

选中一条已发布的推文（特别是 flopped 的），AI 给出「尸检分析」：

```
💀 推文尸检报告 #1234567890

死亡时间：发布后 2 小时
死因：缺乏 hook + 时机不对 + 话题饱和

🔍 详细分析：

时间维度 ❌
  发布于周三 14:00 — 这是科技圈最忙时段，
  你的受众正在开会，没人看手机

内容维度 ❌  
  开头没有 hook — 第 1 句是平淡陈述，
  滑过去的用户 90% 不会再回来

竞争维度 ❌
  同一时间有 3 条相似话题推文，
  其中 @big_influencer 的那条抢走了所有注意力

💡 复活方案：
  1. 改写开头为：「一个大多数人都忽略的事实：...」
  2. 在晚上 21:00 重发（你的受众活跃时段）
  3. 换个时间另起一条，不要直接重发同一条
```

## 为什么有趣

- **学习动机**——每个人都想知道「为什么没人理我」，这比「怎么火」更迫切
- **安慰效果**——尸检报告告诉用户「不是你内容不行，是时机不对」，减轻挫败感
- **幽默感**——「尸检」这个比喻本身就很有梗，适合截图传播

## 技术实现

- 在推文上添加「尸检」按钮（右键菜单或悬浮按钮）
- 提取推文内容、发布时间、互动数据
- 通过 ChatClient 分析多维度因素
- 生成结构化尸检报告

## 交互设计要点

- 推文旁边显示小按钮触发
- 报告用「暗色系」风格，强化尸检主题感
- 可一键分享「我的推文尸检报告」

## 优先级
